E.1.2. 迁移到9.3版本

要从之前的版本迁移数据到这个版本，需要使用pg_dumpall来dump/resotre数据，或者使用pg_upgrade进行升级。

9.3版本包含了一些可能与以前版本不兼容的改变，请注意以下的不兼容问题：

E.1.2.1. 服务器设置

l 重命名replication_timeout为wal_sender_timeout（Amit Kapila）

此设置控制WAL sender超时行为。

l commit_delay参数需要超级用户权限才能设置，因为它现在可能会使其他会话变慢（Simon Riggs）

l 允许内存排序时使用其全部可用内存（Jeff Janes）

用户已设置work_mem需要根据以前的行为重新评估该参数据的设置。

E.1.2.2. 其他

l 过时的元组再次被更新或删除时抛出一个错误（Kevin Grittner）

l ON UPDATE SET NULL / SET DEFAULT外键操作变更为影响到所有引用的列，而不仅仅是那些在UPDATE中引用的列（Tom Lane）

以前只有UPDATE引用的列设置为NULL或默认值。

l 内部存储默认的外键匹配（non-FULL,non-PARTIAL）为“simple”（Tom Lane）

在这之前存储为“<unspecified>”，这改变了系统列pg_constraint.confmatchtype中的值存储。

l 存储WAL为一个连续的数据流，而不是跳过最后每4GB中的最后16MB分部（Heikki Linnakangas）

此前，WAL文件名称中FF结尾的没有被使用。如果你有WAL备份或恢复脚本，考虑了跳过这个文件，那么这些脚本需要调整。

E.1.3. 变化

下面是PostgreSQL 9.3相对之前主要版本变更的详细信息。

E.1.3.1. 服务器

E.1.3.1.1. 锁

l 防止非键字段的更新锁住外键锁（Álvaro Herrera, Noah Misch, Andres Freund, Alexander Shulgin, Marti Raudsepp）

这提高了并发并且降低了死锁的可能性。非键列的更新使用新的SELECT FOR NO KEY UPDATE锁类型，并且外键检查使用SELECT FOR KEY SHARE锁模式。

l 添加配置变量lock_timeout限制锁等待时间（Zoltán Böszörményi）

l 添加本地锁缓存（Jeff Janes）

加速了持有很多锁的事务在完成时释放锁的速度，特别对pg_dump和恢复等操作有用。

E.1.3.1.2. 索引

l 增加了SP-GIST对range数据类型的支持（Alexander Korotkov）

l 允许unlogged的GiST索引（Jeevan Chalke）

l 提高哈希索引的并发性（Robert Haas）

E.1.3.1.3. 优化器

l 收集和使用range类型的直方图（Alexander Korotkov）

l 通过丢弃不必要的低startup costs查询计划，降低优化器的开销（Tom Lane）

l 改善优化器对索引代价的估计（Tom Lane）

E.1.3.1.4. 通用性能优化

l 增加COPY FREEZE选项，避免之后标记记录为已提交的开销（Simon Riggs, Jeff Davis）

l 提高了NUMERIC的计算性能（Kyotaro Horiguchi）

l 改善commit_delay设置时，会话组的等待时间（Peter Geoghegan）

改善了commit_delay参数的实用性和行为。

l 改善了在事务中创建，重建或删除很多表时的性能（Jeff Janes, Tomas Vondra）

l 提升 CREATE TEMPORARY TABLE ... ON COMMIT DELETE ROWS语句的执行性能，现在只有在临时表备访问过时才进行delete（Heikki Linnakangas）

l Vacuum删除过时的记录后进行可见性的重新检查（Pavan Deolasee）

这增加了一个页面被标记为所有都可见的机会。

l 将pg_stat_tmp中的统计文件拆分成每个数据库一个文件和全局文件（Tomas Vondra）

这减少了统计跟踪的I / O开销。

E.1.3.1.5. 监控

l 增加可选的数据页校验能力，可以报告数据的corruption（Simon Riggs, Jeff Davis, Greg Smith, Ants Aasma）

校验选项可以运行initdb的时候被设置。

l 允许同一个角色调用pg_terminate_backend()结束自己另外一个会话（Dan Farina）

在此之前，只有超级用户可以终止会话。

l 允许统计收集器在系统时钟向后推移时正常工作（Tom Lane）

此前统计数据收集的时间会停止，直到再次达到之前统计的最后时间。

E.1.3.1.6. 认证

l 提高LDAP报错信息和文档（Peter Eisentraut）

l 在URL格式中支持LDAP认证（Peter Eisentraut）

l 更改ssl_ciphers参数从DEFAULT开始，而不是ALL，并且移除不安全的密码（Magnus Hagander）

假设DEFAULT是更合适的密码集。

l pg_ident.conf只做一次解析和加载，而不是每个连接都解析和加载（Amit Kapila）

这与pg_hba.conf的处理一致。

E.1.3.1.7. 服务器设置

l 极大地降低了系统V shared_memory的需求（Robert Haas）

相反，使用Unix系统上的mmap()用于共享内存。对于大多数用户来说，这将省去对共享内存内核参数的调整。

l 允许postmaster监听多个Unix域套接字（Honza Horák）

unix_socket_directory这个参数被unix_socket_directories取代，它接受一个目录列表。

l 允许处理一个目录中的多个配置文件（Magnus Hagander, Greg Smith, Selena Deckelmann）

在服务器配置文件中，通过include_dir指定一个目录。

l 将initdb初始化时可以指定的最大shared_buffers参数调整至128MB（Robert Haas）

这是initdb尝试在postgresql.conf中设置的最大值，之前这个最大值为32MB。

l 当postmaster退出时，删除external_pid_file文件（Peter Eisentraut）

E.1.3.2. 复制和恢复

l 允许一个流复制备机跟随主机做timeline转换遵（Heikki Linnakangas）

这允许流复制备机继续从一个被promot的备机上获得流信息。 此slave需要访问WAL归档目录才能实现timeline转换。

l 添加SQL函数pg_is_in_backup()和pg_backup_start_time()（Gilles Darold）

通过这些函数可以获得基础备份的状态信息。

l 提高synchronous_commit disable时流复制日志传送的性能（Andres Freund）

l 允许将流复制备机更快的提升为主机（Simon Riggs, Kyotaro Horiguchi）

l 将最后一个检查点的redo位置增加到pg_controldata的输出中（Fujii Masao）

这个信息对确定恢复需要的WAL文件有用。

l 允许pg_receivexlog运行在不通的计算机架构上（Heikki Linnakangas）

WAL文件仍然可以在相同架构的机器上被重放，但现在它们可以被传送和存储到任意架构的机器上，因为现在流复制协议是架构独立的。

l 使pg_basebackup --write-recovery-conf生成一个最小配置的撇recovery.conf（Zoltán Böszörményi, Magnus Hagander）

这简化了备机的设置。

l 允许pg_receivexlog和pg_basebackup使用--xlog-method来处理流复制时timeline的转换（Heikki Linnakangas）

l 添加wal_receiver_timeout参数控制WAL接收的超时时间（Amit Kapila）

这将允许更快的连接失败检测。

E.1.3.2.1. 预写日志（WAL）

l 更改WAL记录格式使记录头可以跨页（Heikki Linnakangas）

新的格式更紧凑，写入更高效。

E.1.3.3. 查询

l 实现SQL标准中的用在FROM子句中子查询和函数调用的LATERAL选项（Tom Lane）

这个特性允许FROM中的子查询和函数引用FROM中其他表中的列。LATERAL关键字用在函数是是可选的。

l 支持COPY和psql中的\copy读取外部程序重的输出，或将COPY和psql \copy的输出重定向到外部程序（Etsuro Fujita）

E.1.3.4. 对象操作

l 支持event_triggers（Dimitri Fontaine, Robert Haas, Álvaro Herrera）

这允许在DDL命令执行时调用指定的C函数。

l 允许在一个mult-row VALUES字句的规则中引用OLD/NEW（Tom Lane）

l 增加CREATE SCHEMA ... IF NOT EXISTS子句（Fabrízio de Royes Mello）

l 使REASSIGN OWNED改变共享对象的所有权（Álvaro Herrera）

E.1.3.4.1. CREATE TABLE

l 禁止输出有关隐式索引和序列创建的信息（Robert Haas）

这些信息现在出现在DEBUG1级别，所以默认不会显示。

l 允许DROP TABLE IF NOT EXISTS在所指定的模式名称不在时也返回成功（Bruce Momjian）

此前，如果模式不存在，它抛出一个错误。

E.1.3.4.2. 约束

l 向客户端返回constraint violation detail作为单独的字段（Pavel Stehule）

这使得客户端检索表，列的数据类型，或约束名称错误的详细信息。之前，这类信息必须从错误字符串中提取。需要客户端库支持才能访问这些字段。

E.1.3.4.3. ALTER

l ALTER TYPE ...VALUE支持IF NOT EXISTS选项（Andrew Dunstan）

这对测试性的增加枚举类型值非常有用。

l ALTER ROLE ALL SET为所有用户增加设置（Peter Eisentraut）

这允许设置应用于所有数据库中的所有用户。 ALTER DATABASE SET已经允许另外设置一个单一的数据库中的所有用户。 postgresql.conf也有类似的效果。

l 支持ALTERRULE...RENAME（Ali Dar）

E.1.3.4.4. 视图

l 支持物化视图（Kevin Grittner）

不同于每次都要访问基础表的普通的视图，物化视图在创建或者刷新时，会创建物理表。访问物化视图，会从它的物理表中读取。目前还没有任何增量刷新物化视图或自动访问基础表的机制。

l 进行简单的视图自动更新（Dean Rasheed）

引用部分或全部列的简单视图将是可更新的。更复杂的视图，可以使用INSTEAD OF触发器或INSTEAD规则更新。

l 改善视图/规则打印时对引用的表重命名的情况下，或列重命名，添加或删除的处理（Tom Lane）

表名和列重命名可能会给视图/规则带来问题，如果我们仅仅是规则或视图的原始文本替换为新的名称，其结果是有歧义的。该补丁修复了rule-dumping的代码，为了保留原始的语义，插入了表和列的别名。

l 添加CREATE递归视图语法（Peter Eisentraut）

在内部，这被翻译成CREATE VIEW ...WITH RECURSIVE....

E.1.3.5. 数据类型

l 增加large objects的最大长度从2GB到4TB（Nozomi Anzai, Yugo Nagata）

变化主要包括新的libpq和服务器端的64大对象存取功能。

l 允许ISO的“T” timestamptz格式使用文本时区名称如“America/Chicago”（Bruce Momjian）

E.1.3.5.1. JSON

l 添加从JSON字符串中提取数据的运算符和函数（Andrew Dunstan）

l 允许JSON字符串被转换成记录（Andrew Dunstan）

l 添加转换值，记录和hstore到JSON的函数（Andrew Dunstan）

E.1.3.6. 函数

l 增加array_remove()和array_replace()函数（Marco Nenciarini, Gabriele Bartolini）

l 使CONCAT()和format()正确处理不固定参数（Pavel Stehule）

l 改善format()对字段宽度和左/右对齐的处理（Pavel Stehule）

l TO_DATE()和TO_TIMESTAMP()在ISO和Gregorian week/day表示发混用时返回正确的结果（Bruce Momjian）

l 使pg_get_viewdef()默认情况下，在每个SELECT目标和FROM后开始新的一行（Marko Tiikkaja）

这减少了在印刷的线的长度，例如在pg_dump的输出。

l 修复map_sql_value_to_xml_value()，使它打印该函数domain值的输出时按照基础类型的输出方式打印（Pavel Stehule）

某些内置的类型，如布尔，有自己的输出规则;现在，这些规则也适用于对这些类型的domain。

l 如果果SEARCH_PATH的变化，强制已经缓存的函数重新生成查询计划（Tom Lane）

之前，已经在当前会话中运行的函数会忽略SEARCH_PATH变化。
